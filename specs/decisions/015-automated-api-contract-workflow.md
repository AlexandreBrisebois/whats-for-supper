# 015: Automated API Contract Workflow (Kiota & Prism)

## Date
2026-04-23

## Status
Accepted

## Context
The previous API development workflow relied on manual synchronization between the OpenAPI spec, a custom `mock-api.js` file, and manual TypeScript interfaces in the frontend. This created drift, token bloat for agents trying to read massive files, and false positives in testing.

## Decision
1. **Kiota for SDK Generation**: We will use Microsoft Kiota to generate a strictly-typed client SDK directly from `specs/openapi.yaml`. This eliminates manual type management and `fetch` wrapper maintenance.
2. **Prism for Mocking**: We replaced the custom Node.js mock API with Stoplight Prism (`prism mock`). This guarantees 100% parity with the OpenAPI spec automatically.
3. **Reconciliation Tool Updates**: `reconcile_api.py` was updated to recognize Prism, preserving its role as a safety check to ensure the C# Backend remains perfectly aligned with the spec.

## Consequences
- **Positive**: "Contract-First" development is now mechanically enforced. Reduced context window bloat for AI agents. Fewer regression bugs related to API drift.
- **Negative**: The frontend must adapt to the stricter nullable typings generated by Kiota.
